## v1.4.0-beta.3 — 2026-06-28

Rolling beta with **3 additional commits** on top of v1.4.0-beta.2.
Includes one critical `run_as_user` regression fix that blocks 100% of `run_as_user`
users on beta.1/beta.2.

### 🚨 Critical fix

- **claudecode `run_as_user` EACCES regression** — chmod `0o644` on the per-spawn
  system-prompt temp file so non-root child processes can read it after dropping
  privileges. Regression was introduced by v1.3.4 (#1376) when the Windows cmdline
  hotfix switched from inline prompt to file-based prompt with default `0o600` perms.
  Impact: any user running cc-connect with `run_as_user` configured is 100% blocked
  at agent startup with `EACCES` errors. Reported by
  [@vuyiv](https://github.com/vuyiv) (#1429), fixed by
  [@chenhg5](https://github.com/chenhg5) (#1433). +57/-3 across session.go +
  new test `TestWriteTempAppendPromptFile_ReadableByOtherUser`.

### Features

- **Zhipu GLM provider presets** — add `z.ai` and `bigmodel` (CN endpoint) preset
  entries to the embedded `provider-presets.json`. Pure additive — no existing
  presets touched (#1412, [@clingnet](https://github.com/clingnet)).

### Chore

- **`.gitignore`** — add `.worktrees/` for local-only multi-agent scratch dirs.
  Does not affect binary (#1443, [@chenhg5](https://github.com/chenhg5)).

### Why a new beta

beta.2 had QA PASS, but the `run_as_user` regression is a release-blocker for that
config. Out of cycle stable promotion would have shipped this bug — beta.3 is the
right SOP path: re-QA `run_as_user` end-to-end before promotion.
