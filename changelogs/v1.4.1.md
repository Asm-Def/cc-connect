## Fixed

- **Kimi CLI `--print` compatibility** (#1461 fixing #1456,
  [@chenhg5](https://github.com/chenhg5), reported by
  [@WeiFengJL](https://github.com/WeiFengJL))

  Newer `kimi-code` 0.14.x removed the `--print` flag, causing
  `error: unknown option '--print' (Did you mean --prompt?)` on agent
  startup. cc-connect now probes the Kimi CLI `--help` output at startup
  and conditionally passes `--print` only when the installed binary
  supports it. No config change required either way.

  Verified against:
  - `kimi-code` 0.14.2 (probe â†’ `Print=false` â†’ `--print` skipped â†’ unblocked)
  - `kimi-cli` 1.48.0 legacy (probe â†’ `Print=true` â†’ `--print` still passed â†’ no regression)

  Both legacy `kimi-cli` 1.48.x users and new `kimi-code` 0.14.x users
  upgrade to v1.4.1 with zero config changes.
